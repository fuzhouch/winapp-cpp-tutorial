# Hello World App

A simple Windows C++ application demonstrating the use of Microsoft WinAppCLI and CMake.

## Overview

This example shows how to:
- Initialize a project with `winapp init`
- Create a basic Windows application with C++
- Build using CMake
- Run with package identity support

## Prerequisites

- Windows 10/11
- WinAppCLI installed
- CMake 3.10+
- Visual Studio or Build Tools for Visual Studio
- Windows SDK

## Setup

### 1. Initialize with WinAppCLI

This project should be initialized with WinAppCLI to get package identity support:

```bash
cd HelloWorldApp
winapp init
```

Follow the prompts to configure:
- App Name: HelloWorldApp
- Publisher: Your publisher name
- Version: 1.0.0.0
- Framework: C++/Native

This will create:
- `AppxManifest.xml` - App package manifest
- `Assets/` - App icons and assets
- Development certificates

### 2. Build the Project

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

### 3. Run the Application

Option A: Run directly
```bash
./bin/Debug/HelloWorldApp.exe
```

Option B: Run with package identity
```bash
cd ..
winapp create-debug-identity build/bin/Debug/HelloWorldApp.exe
```

## What It Does

This application:
1. Creates a basic Windows window
2. Displays "Hello from WinAppCLI!" text
3. Handles basic window messages (paint, destroy)
4. Demonstrates proper Win32 API usage
5. Works with WinAppCLI package identity

## Extending the App

To add more features:
1. Edit `src/main.cpp` to add UI controls
2. Add more source files to `src/`
3. Add headers to `include/`
4. Update CMakeLists.txt if needed
5. Rebuild with `cmake --build build`

## WinAppCLI Commands

### Restore environment
```bash
winapp restore
```

### Update manifest
```bash
winapp manifest update
```

### Create package
```bash
winapp package create
```

### View logs
```bash
winapp logs
```

## Troubleshooting

**Build errors**: Make sure all prerequisites are installed and CMake can find the Windows SDK.

**Certificate errors**: Regenerate certificate with `winapp certificate create --force`

**Runtime errors**: Try running with `winapp create-debug-identity` to inject package identity.
